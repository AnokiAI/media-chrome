Key Concepts

  • Slots are Placeholders: Components expose named slots that act as placeholders for content. If you don't provide content, the default is used.
  • Override by Name: To override, add an element with slot="slot-name" as a child of the component.
  • Shadow DOM: Slots work with the Shadow DOM, allowing components to encapsulate their implementation while exposing customization points.
  • Multiple Slots: Components can expose multiple slots for different parts (e.g., play icon, pause icon, tooltip).
  • Custom Themes: When building themes, you can define your own custom slots that clients can fill with their content.

How It Works

  1. Define Component: Media Chrome components are defined with slots in their shadow DOM. For example, media-play-button has <slot name="play"> and <slot name="pause">.
  2. Export for Clients: Components are exported as web components that clients can use in their HTML.
  3. Client Integration: Clients import the component and use it in their markup. They can optionally override slots by providing elements with slot="slot-name" attributes.
  4. Rendering: The browser's slot mechanism projects the client's content into the component's shadow DOM at the appropriate slot location.

Common Slots in Media Chrome Components

  • media-play-button: play, pause, icon
  • media-mute-button: mute, unmute, icon
  • media-fullscreen-button: enter, exit, icon
  • media-controller: media, poster, centered-chrome, top-chrome, middle-chrome
  • media-theme: Custom slots you define in your template
